# R² 와 MAPE 의 관계 — 실험과 수학적 정리

> x축 `100 × R²`, y축 `MAPE` 차트에서 출발해, 두 지표의 진짜 관계를
> 수학적 유도와 실제 회귀 실험으로 정리한 문서.

## 0. 한 줄 결론

**R² 와 MAPE 사이에는 보편적인 단일 공식이 없다.** 둘 다 잔차에서 나오므로
"R²↑ → MAPE↓" 의 단조감소 경향은 항상 성립하지만, **정확한 곡선식은
데이터를 어떻게 변동시켰는지(무엇을 고정했는지)에 따라 달라진다.**
직선처럼 보였다면 그것은 생성 방식이 만든 산물이다.

---

## 1. 출발: 직선이 나온 합성 데이터

처음 차트는 데이터를 1차식으로 직접 생성했다.

```
MAPE = 30·(1 − R²) + ε,   ε ~ N(0, 1.5²)
x    = 100·R²
```

### 1.1 왜 직선인가 (증명)

`R² = x/100` 을 대입하면

```
MAPE = 30 − 0.3·x + ε
E[MAPE | x] = 30 − 0.3·x        ← 기울기 −0.3, 절편 30 의 완전한 직선
```

회귀 결과 `y = −0.300x + 30.01` 로 정확히 일치. 즉 **직선은 우연이 아니라
생성식이 1차식이었기 때문**이다.

### 1.2 왜 "거의" 완벽한 직선인가 (상관계수)

`MAPE = a + b·x + ε`, `ε ⊥ x` 이면

```
ρ = b·√Var(x) / √(b²·Var(x) + σ²)
```

`x ~ Uniform(50, 99)` → `Var(x) = (99−50)²/12 ≈ 200.08`, `b = −0.3`, `σ = 1.5`:

```
ρ = −0.3·√200.08 / √(0.09·200.08 + 2.25) ≈ −0.943
ρ² ≈ 0.889   (분산의 ~89% 를 직선이 설명)
```

신호 분산 `b²Var(x) ≈ 18.0` 이 노이즈 분산 `σ² = 2.25` 보다 8배 커서
점들이 직선에 바짝 붙는다. (실측 ρ = −0.936)

**관련 파일:** `r2_vs_mape_chart.py`, `r2_vs_mape_chart.png`

## 2. 실제 회귀모델에서 측정한 진짜 관계

합성식이 아니라 **실제 최소제곱(OLS) 회귀모델**을 사용.
신호를 고정하고 **잡음 σe 만** 바꿔 가며 모델 수천 개를 학습 →
각 모델의 R²·MAPE 를 정의대로 직접 계산.

```
y = 1000 + 8·x₁ − 6·x₂ + 5·x₃ + N(0, σe)
σe: 3 → 115 로 60단계,  각 단계 40회 반복 후 평균 (측정잡음 제거)
```

### 2.1 유도

테스트셋에서 잔차/신호 표준편차를 σe, σs 라 하면

```
1 − R² = σe² / (σs² + σe²)        ⇒   σe = σs·√((1−R²)/R²)
E|y − ŷ| = √(2/π)·σe              (가우시안 잔차)
MAPE ≈ √(2/π)·100·σe / μy
```

대입하면 **단일 파라미터 곡선**:

```
MAPE = K·√((1−R²)/R²),    K = √(2/π)·100·σs / μy
```

(σs, μy 가 고정이므로 K 는 상수 → 모든 점이 한 곡선 위에 놓임)

### 2.2 결과 — 직선이 아니라 곡선

| 모델 | 식 | 적합도 fit-R² |
|---|---|---|
| 순진한 직선 | `MAPE = 7.47 − 0.084·(100R²)` | 0.854 |
| 구조식 곡선 | `MAPE = 2.33·√((1−R²)/R²)` | **0.989** |

- 직선은 양 끝에서 데이터를 체계적으로 벗어나고, R²→100 에서 MAPE 를
  음수로 예측(물리적으로 불가능).
- 이론값 `K = 2.49` 만 넣은 곡선(계수 추정 없이)도 데이터에 잘 붙음
  (적합 K = 2.33, ~6% 오차는 가우시안 근사 편차).

**관련 파일:** `real_r2_vs_mape_experiment.py`, `real_r2_vs_mape_experiment.png`

## 3. 공분산 행렬을 변동시킨 경우

이변량 정규분포를 공분산으로 직접 설계:

```
Σ = s²·[[1, r],
        [r, 1]],    r = √R²
```

- **스케일 s** → 구름 크기: wide(s=3) / narrow(s=1)
- **상관 r** → 타원의 납작함: r=0 이면 원(R²=0), r 클수록 납작한 타원

### 3.1 케이스 (총 9개) 와 측정값

| 케이스 | s | r(목표) | R²(목표) | R²(실측) | MAPE% |
|---|---|---|---|---|---|
| Wide circle    | 3 | 0.000 | 0.0 | 0.001 | 4.66 |
| Wide ellipse   | 3 | 0.447 | 0.2 | 0.215 | 4.12 |
| Wide ellipse   | 3 | 0.707 | 0.5 | 0.479 | 3.46 |
| Wide ellipse   | 3 | 0.837 | 0.7 | 0.721 | 2.53 |
| Narrow circle  | 1 | 0.000 | 0.0 | 0.003 | 1.66 |
| Narrow ellipse | 1 | 0.447 | 0.2 | 0.177 | 1.44 |
| Narrow ellipse | 1 | 0.707 | 0.5 | 0.582 | 1.02 |
| Narrow ellipse | 1 | 0.837 | 0.7 | 0.722 | 0.87 |
| Narrow ellipse | 1 | 0.949 | 0.9 | 0.899 | 0.52 |

(MAPE 정의용으로 Y 에 평균 MU=50 을 더해 양수화 → μy = 50)

### 3.2 관찰

1. **원 vs 타원**: 원(r=0)은 크기와 무관하게 R²=0, 회귀선 수평.
2. **wide vs narrow 는 R² 를 바꾸지 않음**: R² 는 절대 크기가 아니라
   상관 r(타원의 모양)에만 의존. 같은 R² 면 wide·narrow 타원의 납작함은 동일.
3. **공분산 설계 → 실현 상관**: 목표 R² 와 실측 R² 가 y=x 대각선에 일치.

### 3.3 이 경우의 R²–MAPE 곡선

이 설계는 **총분산 s² 고정, 상관 r 만 변동** ⇒ `σe = s·√(1−R²)` ⇒

```
MAPE = C·√(1−R²),    C = √(2/π)·100·s / μy
```

- wide: C = 4.79, narrow: C = 1.60 (s 가 3배 → 곡선 높이 3배)
- 점들이 두 개의 분리된 곡선에 정확히 올라감.

`covariance_r2_matrix.png`, `covariance_r2_correlation.png`, `covariance_r2_mape.png`

## 4. 종합 — 같은 "R²–MAPE" 인데 곡선식이 다른 이유

| 설계 | 고정 | 변동 | 잔차 σe | R²–MAPE 식 |
|---|---|---|---|---|
| §1 합성 | — | (강제 1차식) | — | `MAPE = 30 − 0.3·(100R²)` (직선) |
| §2 회귀실험 | 신호분산 σs² | 잡음 σe | — | `MAPE = K·√((1−R²)/R²)` |
| §3 공분산 | **총분산 s²** | 상관 r | `s·√(1−R²)` | `MAPE = C·√(1−R²)` |

- §1 은 인위적으로 직선을 강제한 것 → 일반 법칙 아님.
- §2, §3 은 모두 **곡선**이며, 분모에 R² 이 있고 없고가 갈린다.
  잡음을 키우면(=§2) 총분산이 함께 커지지만, 총분산을 고정한 채
  설명 비율만 바꾸면(=§3) 다른 곡선이 된다.

**핵심:** R² 와 MAPE 의 관계는 단일 보편식이 아니라,
**데이터 생성/변동 방식이 결정하는 곡선**이다. 정의상으로도 R² 는
제곱오차(L2) 기반, MAPE 는 절대 상대오차(L1) 기반이라 선형 관계가
성립할 본질적 이유가 없다.
